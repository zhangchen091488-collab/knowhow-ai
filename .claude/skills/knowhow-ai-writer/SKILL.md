# KnowHowAI 文档撰写规范

## 概述

本技能指导如何为 KnowHowAI 知识库撰写高质量文档。当用户提供一个知识点、链接或主题时，Agent 应能自动进行调研，并撰写出与知识库风格一致的技术文档。

---

## 文档结构规范

### 标准章节顺序

```
1. 标题 + Frontmatter
2. 简介/概述（为什么需要这个内容）
3. 核心理念/核心概念（用引用块强调关键点）
4. 基础操作/快速开始
5. 进阶内容（可选）
6. 实战示例/代码演示
7. 常见问题 (Q&A)
8. 下一步/相关资源
9. 参考资料
```

### Frontmatter 规范

```yaml
---
title: "序号-emoji 文档标题"
---
```

格式要求：
- `序号`：两位数字，从 01 开始
- `emoji`：与内容相关的图标
- `标题`：简洁的中文名称

**注意**：emoji 已统一为纯文字标题格式（序号-标题），但文件头部的 H1 标题仍保留 emoji

## 写作风格

### 标题层级

- **H2** (`##`)：主要章节，使用 📖 🎯 🚀 等 emoji 前缀
- **H3** (`###`)：子章节，使用 → ⚙️ 🔍 等符号前缀
- **H4** (`####`)：更细分类，极少使用

### 引用块使用

**概念/理念强调**：
```markdown
> **传统工具**是"你问它答"
> **新工具**是"并肩工作"。
```

**关键点提醒**：
```markdown
> 注意：xxx
> 检查 Node 版本：`node --version`
```

### 列表格式

- 无序列表：简洁短句，不以句号结尾
- 有序列表：步骤类内容用 1. 2. 3.

### 表格使用

**对比场景**：
```markdown
| 场景 | 传统模式 | 新模式 |
|------|---------|--------|
```

**规格/要求列表**：
```markdown
| 要求 | 说明 |
|------|------|
```

### Mermaid 图表

知识库广泛使用 Mermaid flowchart 来可视化流程：

```mermaid
flowchart TD
    A[开始] --> B{判断}
    B -->|是| C[执行]
    B -->|否| D[结束]
    
    style A fill:#e3f2fd,stroke:#1976d2
    style C fill:#e8f5e9,stroke:#388e3c
```

常用样式：
- 开始节点：`fill:#e3f2fd,stroke:#1976d2` (蓝色)
- 成功节点：`fill:#e8f5e9,stroke:#388e3c` (绿色)
- 判断节点：`fill:#fff3e0,stroke:#f57c00` (橙色)

## 链接规范

### Wiki 链接命名规则（关键！）

Wiki 链接必须与实际文件名**完全一致**，否则图谱无法正确显示关联！

**正确格式**：
```markdown
[[01-Cursor入门]]           # 文件名：01-Cursor入门.md
[[VibeCoding/02-Skills深度探索]]  # 文件名：02-Skills深度探索.md
```

**常见错误**：
```markdown
# ❌ 错误：链接名与文件名不匹配
[[01-Cursor入门]]            # 实际文件名是：02-Cursor入门.md
[[Cursor入门]]                # 缺少序号
[[02-Skills深度探索]]         # 实际文件名是：03-Skills深度探索.md
```

**检查方法**：
撰写或修改 Wiki 链接后，对比：
1. 链接文字（`[[xxx]]` 中的 `xxx`）
2. 目标文件的实际文件名

### 内部链接

- **同目录 Wiki 链接**：`[[文档名]]`
- **跨目录 Wiki 链接**：`[[分栏名/文档名]]`
- **索引页链接**：`[[分栏名]]` 或 `[[分栏名/_index]]`

### 外部链接

标准 Markdown 格式：
```markdown
[官方文档](https://docs.example.com)
```

## 写作流程

### 1. 调研阶段

- 搜索官方文档
- 查找相关资料和链接
- 理解核心概念和使用场景

### 2. 规划阶段

- 确定属于哪个分栏（VibeCoding / OpenClaw / 效率工具 / Playground）
- 确定序号（检查已有文档，避免重复）
- 选择合适的文档结构模式

### 3. 撰写阶段

- 按标准章节顺序组织内容
- 使用知识库统一的格式和风格
- 添加 Mermaid 图表增强可读性
- 包含实际代码示例

### 4. 自查清单

- [ ] Frontmatter 完整（title 使用序号-emoji-标题 格式）
- [ ] H1 标题包含 emoji
- [ ] 主要章节使用 ## + emoji 格式
- [ ] 代码块标注正确语言
- [ ] 包含 Q&A 或常见问题章节
- [ ] 包含"下一步"指向相关文档
- [ ] 包含参考资料章节
- [ ] 链接格式正确（内部用 wiki 链接，外部用 Markdown）
- [ ] 表格格式正确（表头和数据行格式一致）
- [ ] **Wiki 链接名与实际文件名完全一致**（大小写、编号、扩展名都要匹配）
- [ ] **跨目录链接检查**：链接目标文件必须存在
- [ ] **索引页图谱关联**：新增文档后验证索引页图谱中是否显示该文档节点
- [ ] **避免孤立文档**：每个文档至少被一个其他文档引用（除索引页外）

## 图谱关联故障排查

### 症状：文件夹索引页图谱不显示子文档

**可能原因 1：Wiki 链接名与文件名不匹配**

文件夹索引页（如 `VibeCoding/_index.md`）中的 Wiki 链接必须与实际文件名完全一致。

排查：
```bash
# 查看索引页的链接
grep -o '\[\[[^]]*\]\]' content/分栏名/_index.md

# 查看该目录的实际文件名
ls content/分栏名/*.md
```

**可能原因 2：链接缺少目录前缀**

跨目录链接需要包含完整路径：
```markdown
# ❌ 错误：缺少目录前缀
[[01-Cursor入门]]

# ✅ 正确：包含完整路径
[[VibeCoding/01-Cursor入门]]
```

**可能原因 3：相对路径解析问题**

同目录链接使用短路径，但图谱需要完整路径才能匹配：
```markdown
# 如果文件名是 VibeCoding/02-Cursor入门.md
# ✅ 使用完整文件名（含序号）
[[02-Cursor入门]]
```

**可能原因 4：链接包含 `.html` 后缀**

Quartz 图谱使用 JavaScript 动态解析链接，`.html` 后缀会导致匹配失败：
```markdown
# ❌ 错误：链接中包含 .html 后缀
[文字](文档名.html)

# ✅ 正确：链接中不要包含 .html 后缀
[文字](文档名)
```

### 新增文档后必做检查

1. **索引页链接检查**：新增文档后必须在 `_index.md` 中添加链接，并验证链接名与文件名完全一致
2. **图谱视图验证**：在 `npx quartz build --serve -d content` 预览模式下，打开图谱视图，验证：
   - 新增文档是否出现在相关页面的图谱中
   - 文档之间的双向链接是否正确显示
3. **双向链接验证**：每个文档至少被一个其他文档引用，否则该文档在图谱中是孤立的
4. **避免孤立文档**：如果某个分栏只有索引页链接它，但索引页不显示图谱关联，说明链接格式有问题
